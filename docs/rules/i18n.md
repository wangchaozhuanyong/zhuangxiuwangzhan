# 多语言 i18n 规则

多语言是 Flashcast 的高优先级规则。所有用户看得见的系统文案，都必须走 `src/i18n` 或后台 CMS 多语言字段。

## 强制规则

- 页面和组件里不允许新增硬编码用户可见文案。
- 中文页面不能出现未翻译英文 UI 文案。
- 英文页面不能出现未翻译中文 UI 文案。
- 用户界面禁止直出技术字段，具体见 `docs/rules/user-facing-technical-fields.md`。
- `status`、`event_type`、`source`、`error_code`、`message` 等字段进入页面前必须映射成当前语言文案。
- `Loading`、`Submit`、`Read more`、`Save`、`Delete` 这类系统文案必须走翻译。
- 品牌名、WhatsApp、API、SEO、URL、技术名可以保留英文，但必须符合 `i18n-locale-rules.json` 的 allowlist 规则。
- SEO 文案、按钮文案、表单提示、错误提示、空状态、loading 状态都属于 i18n 范围。
- 后台 CMS 内容必须考虑中文和英文字段，不能只补中文、不补英文。

## 后台和 CMS

- 系统 UI 文案走 `src/i18n`。
- 后台录入的服务、案例、材料、博客、FAQ、落地页、SEO 内容走 CMS 多语言字段。
- 新增 CMS 字段时，必须检查中文字段、英文字段、前台读取、后台编辑器和 SEO 输出是否一致。
- 英文生成或翻译任务不能覆盖已有英文内容，除非任务明确要求。

## 修改后检查

涉及多语言改动时必须检查：

- 是否新增了翻译 key。
- 中文和英文是否都补齐。
- 是否误用了 fallback 文案。
- 是否还有 `snake_case`、`kebab-case`、英文系统句子直接出现在用户界面。
- 是否需要更新 `i18n-locale-rules.json`。
- 是否运行或说明未运行 `npm run i18n:check`。

## 禁止事项

- 禁止在中文页留下未确认的英文 UI 文案。
- 禁止在英文页留下中文 UI 文案。
- 禁止把 `system_health_check`、`permission_denied`、`upload_failed` 这类技术字段直接展示给用户。
- 禁止为了让检查通过而把真实需要翻译的文案加入 allowlist。
- 禁止只改前台文案，不检查后台和 SEO 文案。
