# 可访问性和响应式规则

公开页面和后台关键页面都必须保证基本可访问性。Flashcast 的前台页面还必须满足“无重叠、无横向溢出、无逻辑冲突”的验收标准。

## 响应式至少检查

- 360px 手机宽度。
- 390px 常见手机宽度。
- 768px 平板宽度。
- 1024px 小桌面宽度。
- 1440px 大屏宽度。

## 前台页面必须检查

- 导航不能遮挡正文。
- 文案不能重叠。
- 页面不能横向溢出。
- 卡片、按钮、表格、图片不能挤破容器。
- 移动端底部行动栏不能遮挡关键内容。
- 中文和英文长文案都不能撑爆布局。
- 页面不能出现 `�` 或明显乱码。

## 可访问性要求

- 图片必须有有意义的 `alt`，装饰图可用空 `alt`。
- 表单字段必须有 `label` 或清楚的无障碍名称。
- 按钮必须有清楚文字或 `aria-label`。
- 图标按钮必须能让用户知道用途。
- 错误状态不能只靠颜色表达。
- 弹窗、下拉、菜单打开后焦点不能乱跑。
- 页面必须保留跳过导航或主要内容可聚焦能力。
- 键盘能操作的地方不能被 CSS 或脚本破坏。

## 验证建议

- 前台 UI 改动优先用真实浏览器或 Playwright 检查桌面端和移动端。
- 表单页面要检查成功、失败、校验错误、loading 和移动端状态。
- 后台页面至少要保证移动端不遮挡、不横向溢出到不可操作。

## 禁止事项

- 禁止只看 1440px 桌面端就宣称完成。
- 禁止只靠颜色表达错误、禁用、成功状态。
- 禁止按钮只有图标但没有可读名称。
- 禁止让弹窗打开后键盘焦点丢失。
