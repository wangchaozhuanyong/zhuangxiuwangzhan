# 依赖管理规则

新增依赖会影响包体积、构建、维护和安全。Flashcast 不允许为了方便随手加包。

## 新增依赖前必须说明

- 为什么需要它。
- 它解决什么具体问题。
- 项目已有依赖是否已经能解决。
- 是否有更轻量方案。
- 是否影响包体积、构建速度、浏览器兼容或部署。
- 是否活跃维护、是否有安全风险。

## 高风险依赖

以下依赖类型默认需要更谨慎：

- 大型 UI 组件库。
- 状态管理库。
- 表格库。
- 图表库。
- 富文本编辑器。
- 拖拽/低代码/页面编辑器。
- 支付、认证、上传、安全相关 SDK。

## 修改后检查

- 新增依赖后必须同步 `package-lock.json`。
- 至少运行安装、类型检查或构建中的相关命令。
- 如果依赖影响前台包体积，应说明影响范围。
- 如果依赖影响部署环境，应说明 Node、Cloudflare、Supabase Edge Function 兼容性。

## 禁止事项

- 禁止为一个小功能引入一个大依赖。
- 禁止引入和现有设计系统冲突的组件库。
- 禁止安装后才解释为什么需要。
- 禁止只改 `package.json` 不更新 lockfile。
